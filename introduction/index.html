Haskell

Purely functional

  • Functions only take arguments and return values
  • Data is immutable
  • Functions are first-class citizens

Advantages

  • Modularity
  • Abstraction
  • Testing is a breeze
  • Good for concurrency
  • Allows better optimization by compiler

Lazy

  • Computations deferred until their result is actually needed
  • Allows for infinite data structures
  • Better separation of concerns

Statically typed

  • Strong type system
  • Everything is a value, every value has a type
  • Catches a lot of errors
  • Types are automatically inferred
  • Serves as documentation

Batteries included

  • Haskell Platform
  • Easy to install and get started

Libraries

  • No shortage of high quality libraries
  • Hackage is the repository for Haskell packages
  • Packages are easy to install
  • There's a package for just about anything

Haskell is fun

  • Possibly more than rollercoasters
  • Think differently about problems
  • Play around in the interactive mode

Super nice community

  • Friendly to newbies
  • Great representation on Stack Overflow
  • Come hang out in #haskell

Analyzing character frequency

λ> let sentence = "whoa there man"

λ> let letters = [c | c <- sentence, c /= ' ']
λ> letters
"whoathereman"

λ> sort letters
"aaeehhmnortw"

λ> group (sort letters)
["aa","ee","hh","m","n","o","r","t","w"]

λ> [(length xs, head xs) | xs <- group (sort letters)]
[(2,'a'),(2,'e'),(2,'h'),(1,'m'),(1,'n'),(1,'o'),(1,'r'),(1,'t'),(1,'w')]

Let's make a function out of this

import Data.List

frequency :: [Char] -> [(Int, Char)]
frequency sentence = [(length xs, head xs) | xs <- groups]
    where letters = [c | c <- sentence, c /= ' ']
          groups = group (sort letters)

λ> frequency "asdksjdhfwasl"
[(2,'a'),(2,'d'),(1,'f'),(1,'h'),(1,'j'),(1,'k'),(1,'l'),(3,'s'),(1,'w')]

Recursion

  • No modifying of state
  • So no for loops
  • Use recursion to iterate
sum' :: [Int] -> Int
sum' [] = 0
sum' (x:xs) = x + sum' xs

sum' [5,4,8,1]                =
5 + sum' [4,8,1]              =
5 + (4 + sum' [8,1])          =
5 + (4 + (8 + sum' [1]))      =
5 + (4 + (8 + (1 + sum' []))) =
5 + (4 + (8 + (1 + 0)))

Thinking inductively

  1. Solve trivial example (the base case)
  2. Pretend our function works
  3. Split into subproblems
  4. Solve subproblems by using our function, then put together final solution
reverse' :: [a] -> [a]
reverse' [] = []
reverse' (x:xs) = reverse xs ++ [x]

reverse' [1,0,3,5]                           =
reverse' [0,3,5] ++ [1]                      =
(reverse' [3,5] ++ [0]) ++ [1]               =
((reverse [5] ++ [3]) ++ [0]) ++ [1]         =
(((reverse [] ++ [5]) ++ [3]) ++ [0]) ++ [1] =
((([] ++ [5]) ++ [3]) ++ [0]) ++ [1]         =
[5,3,0,1]

Folds

  • Many recursive functions that operate on lists follow the same pattern
  • Base case for the empty list
  • We repeatedly apply some function based on the first element and the rest of the list
  • Folds allow us to reduce a list to a single value
sum xs = foldl (+) 0 xs
  • foldl folds up a list from the left
  • It takes a binary function, a starting accumulator, and a list to fold up
  • Repeatedly applies the function on the starting accumulator and an element in the list, resulting in a new accumulator
  • When traverses all the elements, it returns the accumulator
foldl (+) 0 [6,3,7,1]

0 +  6
    [6,3,7,1]

6 +  3
    [3,7,1]

10 + 7
    [7,1]

17 + 1
    [1]

18

Folds

foldl f z [a,b,c,d] = f (f (f (f z a) b) c) d)

foldr f z [a,b,c,d] = f a (f b (f c (f d z)))

product xs = foldl (*) 1 xs

any xs = foldl (||) False xs

map f xs = foldr (\x acc -> f x:acc) [] xs

filter f xs = foldr (\x acc -> if f x then f:acc else acc) [] xs

An RPN calculator

  • Let's make a function that solves algebraic expressions in reverse polish notation
  • Usually operators are infix: 10 * (5 + 3)
  • Instead of putting the operator between its operands (4 + 3) we can put it after them: 4 3 +
  • 4 and 3 go on a stack and then + operates on them because they're at the top of the stack
  • 4 3 + 7 * is equivalent to (4 + 3) * 7

We'll take a string, like this:

We go over it left from right and keep track of a stack as we do that

An RPN calculator

solveRPN :: String -> Double
solveRPN exp = head (foldl foldingFunction [] (words exp))
    where foldingFunction (x:y:ys) "*" = (y * x):ys
          foldingFunction (x:y:ys) "+" = (y + x):ys
          foldingFunction (x:y:ys) "-" = (y - x):ys
          foldingFunction xs numberString = 
            read numberString:xs

λ> solveRPN "10 4 3 + 2 * -"
-4.0
λ> solveRPN "4 3 +"
7.0
λ> solveRPN " 3 8 2 9 * + -"
-23.0
